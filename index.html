SmartOps (AAOS)

Adaptive Accelerator Orchestration System

View Live Demo

"Autonomous GPU resource optimization for hybrid AI pipelines."

SmartOps is a simulation engine designed to solve the "Static Configuration Problem" in ML infrastructure. It acts as a self-healing control plane, continuously monitoring hardware telemetry (VRAM, Latency, Bandwidth) to autonomously adjust pipeline hyperparameters in real-time. It eliminates manual tuning, prevents OOM crashes, and ensures strict latency SLAs.

Key Technical Features

  • The "Brain" (Heuristic Policy Engine): A rule-based system that dynamically scales resources. It slashes physical batch sizes during latency spikes and boosts prefetch factors during data starvation.
  • OOM Prevention & Recovery: Detects critical memory pressure (>95% VRAM) and triggers "defense modes," such as switching precision (FP32 → FP16) or decoupling physical from virtual batch sizes.
  • Unified Priority Scheduler: Simulates a multi-tenant environment managing heterogeneous workloads (LLMs, Vision, Audio). It uses priority queuing to arbitrate resource contention.

LatentFuse v2

Efficient Sparse Multimodal Attention

View Live Demo

"Proving multimodal efficiency through real-time GFLOPs profiling."

Standard multimodal models like CLIP are computationally expensive. LatentFuse v2 addresses this by using efficient backbones (ResNet18 & DistilBert) and a novel sparse latent attention mechanism. Instead of attending to every token, a small set of learnable "latent queries" attend only to the top-k most relevant image and text features, decoupling computational cost from input size.

Key Technical Features

  • Live Architecture Profiling: The Gradio demo instantiates the actual PyTorch model and uses fvcore to measure GFLOPs and parameter count in real-time, proving efficiency against a CLIP baseline.
  • Sparse Latent Attention: Implemented a custom attention module where fixed, learnable query vectors attend to a pruned selection (top-k) of input tokens, significantly reducing attention map size.
  • Hybrid Backbone Design: Combines the inductive biases of CNNs for images (ResNet18 truncated) with the contextual power of Transformers for text (DistilBert) for an optimal speed/accuracy trade-off.

Website & PDF QA Bot

Universal Retrieval-Augmented Generation (RAG)

View Live Demo

"Chat with any document or website instantly."

This project bridges the gap between static knowledge (PDFs) and dynamic information (Websites) by creating a unified interface for questioning unstructured data. It utilizes a full RAG pipeline to ingest data, generate embeddings, and retrieve context for the Mixtral-8x7B LLM, ensuring answers are grounded in the provided source material rather than model hallucinations.

Key Technical Features

  • Dual-Source Ingestion: Built custom loaders using PyMuPDFLoader for complex PDFs and WebBaseLoader for scraping live websites, merging them into a single vector index.
  • Vector Search with FAISS: Implemented high-speed similarity search using FAISS (Facebook AI Similarity Search) and sentence-transformers/all-MiniLM-L6-v2 embeddings to retrieve the most relevant context chunks.
  • Mixtral-8x7B Integration: Leveraged the power of Mixture-of-Experts (MoE) via Hugging Face Hub inference API to provide high-quality, nuanced answers with low latency.

2-Stage LLM Guardrails

Real-time Safety Filtering & Moderation

View Live Demo

"Ensuring enterprise AI safety through a multi-layered defense system."

This project demonstrates a robust approach to securing LLM interactions. Instead of relying solely on prompt engineering, it implements a deterministic, multi-stage "firewall" that sits between the user and the model. It proactively detects and blocks adversarial inputs (jailbreaks) and filters toxic outputs in real-time.

Key Technical Features

  • Semantic Jailbreak Detection: Utilizes sentence-transformers to embed user prompts and compare them against a vector database of known malicious goals from the JailbreakBench dataset. High cosine similarity scores trigger an immediate block.
  • BERT-Based Toxic Content Filtering: Integrated the unitary/toxic-bert model to classify the tone of both inputs and outputs. It flags content for categories like threats, insults, and hate speech with high precision.
  • Heuristic & PII Checks: Implemented a foundational layer of regex-based checks to instantly catch PII leaks (like email addresses) and explicit banned keywords before more expensive model-based checks run.

MetaGlimpse

Automated Metadata & Profiling Engine

View Live Demo

"Instant dataset diagnostics and feature importance analysis."

MetaGlimpse is a "Dataset-Agnostic" analysis tool designed to solve the "cold start" problem in Data Science. It instantly ingests raw CSVs and outputs a standardized JSON schema describing quality, cardinality, and missingness, while automatically calculating target-aware feature importance without manual coding.

Key Technical Features

  • Lazy-Loaded Architecture: Implemented deferred importing for heavy libraries (YData Profiling) to ensure instant app startup and optimize memory usage on cloud runtimes.
  • Hybrid Importance Calculation: Dynamically selects between Mutual Information (entropy-based) and Pearson Correlation based on whether the target variable is detected as Categorical or Continuous.
  • JSON-First Output: Generates machine-readable metadata schemas, enabling this tool to act as a pre-processing step for larger AutoML pipelines.
